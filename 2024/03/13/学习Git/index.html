指针

Git的每个版本都由指针指向（也可以用唯一的id标识），当前版本指针为HEAD ；HEAD~ 和HEAD^ 表示上一个版本的指针；HEAD~n表示前 n 个版本的指针。

分支

分支可以独立存在，并且有自己的提交记录。分支非常适合团队协作和开发管理。多个开发人员可以在自己的分支上进行开发工作，最后再合并到主线上。这样就能保证主线代码仓库处于一个随时可用的稳定状态，而不会影响其他功能的开发和测试，保证了项目的正常运行和高效协作。

分支能够提高团队协作的效率，减少冲突和错误的影响，让团队中每个人都能独立开发和测试。

如果两个分支同时修改了同一个文件的同一行代码，那么就会产生冲突。Git在合并分支的时候不知道保留哪一个代码，这就是需要我们手动解决冲突。

发送冲突后，冲突的文件会显示两个文件的内容，所有需要手动删除某些内容，然后再git add 和 git commit ；或者使用git merge --abort

变基

变基（rebase）可以将两个分支拼接成一个分支，例如在main分支上执行git rebase dev ,那么就会将main分支和dev分支共同祖先之后的提交记录拼接到dev分支之后，然后两个分支汇成一个分支

变基（rebase）和合并（merge）的区别：

merge：

  • 优点：不会破坏原分支的提交历史，方便回溯和查看

  • 缺点：会产生额外的提交节点，分支图比较复杂

    rebase：

  • 优点：不会新增额外的提交记录，形成线性历史，比较直观和干净

  • 缺点：会改变提交历史，改变了当前分支branch out的节点，避免在共享分支使用。
