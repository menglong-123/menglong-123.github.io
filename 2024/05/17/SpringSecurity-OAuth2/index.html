SpringSecurity & OAuth2

SpringSecurity：一个安全性框架 官方文档：https://spring.io/projects/spring-security

OAuth2：开发标准的授权协议

SpringSecurity的功能：

  • 身份认证

    • 验证谁正在访问系统资源，判断用户是否为合法用户。
    • 认证是用来保护系统资源
    • 认证用户的常见方式是要求用户输入用户名和密码

  • 授权

    • 用户进行身份验证后，系统会控制谁能访问那些资源，这个过程叫授权。用户无法访问没有权限的资源。

  • 防御常见攻击

    • CORS
    • HTTP Headers
    • HTTP Requests

有两种常用的认证方式：

  • 基于Session

    • 用户使用账号密码或其他方式认证成功后，服务端会生成一个Session来保存用户数据，然后将Session id发给客户端，客户端将其存放到Cookie中，以后每次请求都携带session id
    • 基于Session的方式，服务端要保存该Session信息，会话保存在session中

  • 基于token

    • 用户使用账号密码或其他方式认证成功后，服务端会生成一个token来唯一标识一个用户，然后将token返回给客户端，客户端可以放到cookie或localStorage中，以后每次请求都携带token
    • 基于token的方式，服务端可以不保存token，用户信息都保存在token中，会话保持在token中

SpringSecurity默认做了什么？

  • 保护应用程序URL，要求对应用程序的任何交互进行身份校验
  • 程序启动时生成一个默认的用户user，和默认的随机密码（输出在控制台上）
  • 生成默认的登录表单和注销页面
  • 提供基于表单的登录和注册流程
  • 对于Web请求，重定向到登录页面
  • 对应服务请求，返回401未经授权
  • 处理跨站请求伪造(CSRF)攻击
  • 处理会话劫持攻击
  • 写入Strict-Transport-Security以确保HTTPS
  • 写入X-Content-Type-Options以处理嗅探攻击
  • 写入Cache Control头来保护经过身份验证的资源
  • 写入X-Frame-Options以处理点击劫持攻击。
