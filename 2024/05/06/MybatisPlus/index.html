MybatisPlus

MybatisPlus是Mybatis的增强。MybatisPlus的特点：

  • 无侵入式：只做增强不做改变，引入它不会对现有工程产生影响
  • 只需要简单的配置，就能快速对单表CRUD操作
  • 具有代码生成、自动分页、逻辑删除、自动填充等功能。

官方文档：https://baomidou.com/

使用MybatisPlus

  • 引依赖
  • 定义mapper, 自定义的mapper继承与MybatisPlus提供的BaseMapper接口
  • 直接使用BaseMapper中的方法来实现单表操作

MybatisPlus的约定

MybatisPlus通过扫描实体类，并基于反射获取实体类信息作为数据库表信息：

默认使用类名的驼峰转下划线作为表名，名为id的为主键，变量名驼峰转下划线作为表的字段名

MybatisPlus的注解

若数据库表和实体类的映射不满足约定，那么就需要使用注解来自定义。MybatisPlus常见的注解：

  • @TableName：用来指定表明

  • @TableId：用来指定表中的主键字段信息，type字段来指定主键的生成策略。IdType枚举

    • AUTO：数据库自增
    • NONE：表示未设置主键类型（注解里等于跟随全局,全局里约等于 INPUT）
    • INPUT：通过set方法自行输入
    • ASSIGN_ID：分配ID，接口IdentifierGenerator的方法nextId来生成id，默认实现类为DefaultIdentifierGenerator雪花算法
    • ASSIGN_UUID：通过UUID分配ID
    • 默认使用雪花算法

  • @TableField：用来指定表中的普通字段信息

    • 成员变量名和数据库字段名不一致
    • 成员变量名以is开头，且是布尔值
    • 成员变量名与数据库字段名冲突（@TableField(“order“)）
    • 成员变量名不是数据库字段（@TableField(exist=false))）

MybatisPlus静态工具

MybatisPlus的Db类中有各种静态方法，能够执行增删改查。这些静态方法和IService中的方法相似，只不过继承IService接口时要指定实体类泛型，而Db静态类在使用方法是要指定实体类的Class对象，然后通过反射来执行。若静态方法的参数有实体类信息时，就不需要传它的class对象了。

例如：User user = Db.getById(1L, User.class);

静态工具的使用情景：

若只使用IService接口的话，可能存在以下情景：Service A中需要注入Service B，Service B中也需要注入Service A，那么就会产生循环依赖。

使用静态工具类中的方法就不用注入Service，因此也不会出现循环依赖的问题。

逻辑删除

逻辑删除是基于代码逻辑模拟删除效果，并不会真正删除数据，思路如下：

  • 在表中添加一个字段标记数据是否删除
  • 当删除数据时把标记数据置为1
  • 查询是只查询标记为0的数据

删除操作： update user set deleted = 1 where id = 1 and deleted = 0
查询操作： select * from user where deleted = 0

MybatisPlus提供了逻辑删除的功能，无需改变方法的调用方式，而是在底层帮我们自动修改CRUD语句。我们只需在application.yaml文件中配置逻辑删除的字段名称和值即可。

或者在实体类上加注解@TableLogic

使用逻辑删除存在的问题

  • 导致数据库表中垃圾数据越来越多，影响查询效率
  • SQL语句中都要加逻辑删除字段判断，影响查询效率

因此逻辑删除不太推荐，可以采用将删除数据迁移到其他表（也有其他问题）

枚举处理器

在数据库表中，如果有需要的状态字段，通常会定义为int类型，若Java实体类也采用int类型，在其他地方使用该状态会导致代码的可读性不好，因为可以使用Mybatis提供的枚举处理器。
使用枚举处理的流程

  1. 定义枚举，使用@EnumValue来标注哪个属性是枚举的值
  2. 将实体类中该字段的类型由Integer转为UserStatus类型
  3. 在application.yaml配置文件中配置枚举处理器 这样在Java代码其他地方就能直接使用枚举类型来表示该字段的值，同时MybatisPlus也能成功解析出枚举对应的值来和数据库做交互。

Json处理器

若实体类中有一个String类型的Json字段与数据库对应，那么我们可以使用MybatisPlus提供的Json处理器。

使用Json处理器

  • 修改改字段的类型，由String改为实体类类型

  • 在该字段上添加注解, @TableField(typeHandler = JacksonTypeHandler.class)

  • 在实体类上添加注解, @TableName(value = "user", autoResultMap = true)

这样就能将实体类转化为json字符串存储到数据库中。
